## 概述

本文档定义EAS系统的分层架构（C4 Level 2），展示容器级别的系统组成与交互关系。

## 容器说明

### 前端容器

| 容器 | 技术栈 | 说明 |
|-----|--------|------|
| Web前端 | Vue 3 + Vite + Pinia | 员工/HR使用的主界面 |
| Mobile App | React Native | 移动端应用 |
| 管理后台 | Vue 3 + Element Plus | 管理员配置界面 |

### 后端服务容器

| 容器 | 端口 | 数据库 | 职责 |
|-----|------|--------|------|
| eas-hr-context | 8080 | hr_db | 人力资源、招聘、培训、考勤 |
| eas-pm-context | 8081 | pm_db | 项目、任务、Issue管理 |
| eas-crm-context | 8082 | crm_db | 客户、商机、合同、回款 |
| eas-org-context | 8083 | org_db | 组织架构、部门、岗位 |
| eas-auth-context | 8084 | auth_db | 认证、授权、JWT |
| eas-notification-context | 8085 | notification_db | 消息通知、模板 |

### 基础设施容器

| 容器 | 版本 | 说明 |
|-----|------|------|
| MySQL | 8.0 | 关系数据库 |
| RabbitMQ | 3.12 | 消息队列 |
| Redis | 7.0 | 缓存、会话 |
| Nginx | 1.24 | 反向代理、静态资源 |
| MinIO | Latest | 对象存储（可选） |

## 服务间通信

### 同步通信 (REST)

```
前端 → Nginx → 各上下文服务

PM上下文 → HR上下文 (ACL调用)
PM上下文 → CRM上下文 (ACL调用)
各上下文 → 组织上下文 (ACL调用)
各上下文 → 认证上下文 (Token验证)
```

### 异步通信 (消息队列)

```
CRM上下文 --[合同签订事件]--> PM上下文
PM上下文 --[人员释放事件]--> HR上下文
各上下文 --[通知事件]--> 通知上下文
```

## 部署架构

### 开发环境

使用Docker Compose一键启动：

```yaml

services:
  eas-hr: ...
  eas-pm: ...
  eas-crm: ...
  eas-org: ...
  eas-auth: ...
  eas-notification: ...
  mysql: ...
  rabbitmq: ...
  redis: ...
  nginx: ...
```

| 选项 | 说明 |

## 数据库分离

每个限界上下文使用独立的数据库：

| 数据库 | 上下文 | 说明 |
|-------|--------|------|
| hr_db | HR上下文 | 员工、招聘、培训、考勤 |
| pm_db | PM上下文 | 项目、任务、Issue |
| crm_db | CRM上下文 | 客户、商机、合同 |
| org_db | 组织上下文 | 组织、部门、岗位 |
| auth_db | 认证上下文 | 用户、角色、权限 |
| notification_db | 通知上下文 | 通知、模板 |

这是DDD的重要原则：**每个限界上下文拥有自己的数据存储**，避免跨库Join。
