# EAS 系统上下文

> 本文档记录 EAS 系统的系统上下文定义。
> 来源：《解构领域驱动设计》第 20.3.3 节第 1 小节
> 对应图：图20-27

---

## 系统定义

EAS（Enterprise Administration System）是目标系统，整个系统上下文以EAS为中心。

---

## 用户

参与系统上下文的用户包括：

| 用户角色 | 说明 |
|---------|------|
| 集团决策者 | 查看统计报表，进行管理决策 |
| 市场部 | 管理市场需求、客户需求、订单 |
| 人力资源部 | 员工管理、招聘、培训 |
| 项目管理部 | 项目立项、进度跟踪、问题管理 |
| 子公司 | 项目执行、资源管理 |
| 服务中心 | 硬件资源分配 |
| 财务 | 财务核算、合同管理 |
| 员工 | 工作日志、考勤、培训参与 |

---

## 伴生系统

### OA 系统

- **关系**：EAS 的伴生系统
- **功能**：提供部门之间的流程协作与消息通知
- **说明**：集团已有OA系统，EAS不需要重复开发流程协作与消息通知功能，但需要与之集成

### 薪资管理系统

- **关系**：EAS 的伴生系统
- **功能**：提供员工薪资接口
- **说明**：客户合作流程服务蓝图中的内部支持者包含薪资管理系统，调用它提供的服务接口可以获得员工薪资，以便财务进行财务核算

---

## 非伴生系统

### 软件学院与招聘网站

- **关系**：非伴生系统（不集成）
- **说明**：系统当前状态虽然告知软件学院和招聘网站的简历会作为集团的人才储备库，但通过招聘流程服务蓝图可以确知，这些简历信息需要招聘专员手工录入EAS系统。因此，EAS系统的伴生系统并不包含软件学院和招聘网站

## 小结

确定了系统上下文，就确定了EAS系统的解空间，同时也确定了位于解空间之外的伴生系统。系统上下文为后续的限界上下文识别和上下文映射提供了边界约束。
