## 外部用户

| 用户类型 | 主要职责 | 使用场景 |
|---------|---------|---------|
| HR经理 | 人力资源决策 | 员工管理、招聘审批、组织规划 |
| HR专员 | 人力资源执行 | 员工入职离职、考勤管理、培训组织 |
| PM总监 | 项目组合管理 | 项目立项审批、资源调配决策 |
| 项目经理 | 项目执行管理 | 任务分配、进度跟踪、团队协调 |
| 销售总监 | 销售管理 | 商机跟进决策、合同审批 |
| 销售经理 | 客户关系维护 | 客户拜访、商机推进、回款跟进 |
| 财务总监 | 财务决策 | 合同审核、回款认领、开票审批 |
| 薪资专员 | 薪资核算 | 考勤数据导入、绩效数据接收 |
| 财务专员 | 财务执行 | 开票申请、回款登记 |
| 员工 | 日常操作 | 考勤打卡、培训报名、任务更新 |

## 伴生系统

### OA系统 (Organization Automation)

| 属性 | 说明 |
|-----|------|
| 系统类型 | 上游数据源 |
| 集成方式 | REST API + 消息队列 (RabbitMQ) |
| 交互内容 | 组织架构变更、员工入职离职事件、审批流程 |
| 集成频率 | 实时事件 + 定时全量同步(每日) |
| 数据流向 | OA → EAS (主要) |
| 备注 | OA维护组织架构权威数据源 |

### 薪资管理系统 (Payroll System)

| 属性 | 说明 |
|-----|------|
| 系统类型 | 下游消费方 |
| 集成方式 | REST API + 文件上传(FTP/SFTP) |
| 交互内容 | 考勤数据、绩效数据、调薪生效事件 |
| 集成频率 | 月度批量推送 |
| 数据流向 | EAS → 薪资系统 |
| 备注 | 薪资系统作为独立系统存在 |

### 财务系统 (Financial System)

| 属性 | 说明 |
|-----|------|
| 系统类型 | 下游消费方 |
| 集成方式 | REST API |
| 交互内容 | 合同信息推送、开票申请、回款认领 |
| 集成频率 | 实时触发 |
| 数据流向 | 双向 (EAS → 财务, 财务 → EAS) |
| 备注 | 需要支持开票状态回调、回款确认 |

## 集成接口定义

### OA → EAS 接口

```
事件消息格式 (RabbitMQ):
{
  "eventType": "ORG_CREATED" | "ORG_UPDATED" | "ORG_DELETED" |
               "EMPLOYEE_HIRED" | "EMPLOYEE_TRANSFERRED" | "EMPLOYEE_RESIGNED",
  "eventTime": "2026-05-24T10:30:00Z",
  "entityId": "org-12345",
  "payload": { ... }
}
```

### EAS → 薪资系统 接口

```
POST /api/payroll/sync-attendance
Request:
{
  "month": "2026-05",
  "records": [
    {"employeeId": "E001", "workDays": 22, "leaveDays": 0, "overtimeHours": 5}
  ]
}
```

### EAS ↔ 财务系统 接口

```
POST /api/finance/contracts
Request: {合同基本信息}

POST /api/finance/invoices
Request: {开票申请}

POST /api/finance/payments/confirm
Request: {回款确认}

GET /api/finance/payments/{contractId}
Response: {回款状态}
```

## 边界决策

### 不纳入EAS的功能

| 功能 | 决策原因 | 替代方案 |
|-----|---------|---------|
| 请假审批 | OA已有成熟流程 | 通过OA审批，EAS记录结果 |
| 公文流转 | 非核心业务 | 继续使用OA |
| 薪资计算 | 专业性强、法律风险高 | 数据推送到专业薪资系统 |
| 开票打印 | 财务系统功能 | EAS触发申请，财务系统执行 |
| 财务核算 | 需符合会计准则 | 专业财务系统处理 |
