### 2. 架构图评审

大家对系统上下文图和上下文映射图进行了评审。

**王强提出**：PM作为协作中心，会不会成为性能瓶颈？

**张毅回应**：
1. PM的ACL调用可以考虑加缓存
2. 非核心路径可以降级（比如合同信息可以异步加载）
3. 真有瓶颈再说，30天项目先做出来

**孙磊提出**：CRM和PM的"项目"概念不同，会不会在联调时混乱？

**张毅回应**：
1. 我们已经在统一语言词典里明确标注了
2. API文档里要明确标注所属上下文
3. 代码包名隔离：com.eas.pm.project vs com.eas.crm.project

### 3. 技术选型确认

| 技术选型 | 决策 | 理由 |
|---------|------|------|
| ORM | MyBatis | SQL可控，性能优于JPA |
| API Gateway | 不使用 | 30天项目，Spring Boot DispatcherServlet足够 |
| 服务发现 | 不使用 | 固定端口，Nginx反向代理 |
| 配置中心 | 不使用 | 配置文件+环境变量 |
