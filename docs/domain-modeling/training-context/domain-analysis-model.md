# 培训上下文领域分析模型

> 来源：《解构领域驱动设计》第 20.3.4 节 — 图 20-51
> 方法：快速建模法，以"提名候选人"业务服务规约 EAS-0202 为入口

---

## 1. 领域概念识别

### 1.1 从名词提取的候选概念

从"提名候选人"业务服务规约的基本流程中提取名词：

| 名词（中文） | 英文 | 来源 |
|-------------|------|------|
| 候选人 | Candidate | 业务服务名、基本流程 |
| 协调者 | Coordinator | 业务服务描述 |
| 培训 | Training | 业务领域核心概念 |
| 课程 | Course | 基本流程"确定候选人是否已经参加过该课程" |
| 票 | Ticket | 统一语言——系统将分配给员工的培训机会称为"票" |
| 候选名单 | CandidateList | 业务流程中"属于候选名单中的员工" |
| 票状态 | TicketStatus | 验收标准"提名的票状态必须为 Available" |
| 邮件 | Mail | 基本流程"邮件通知获得提名的候选人" |

### 1.2 从动词发现的隐藏概念

| 动词短语 | 隐藏概念 | 英文 | 理由 |
|---------|---------|------|------|
| 确定候选人是否已经参加过该课程 | 学习记录 | Learning | 系统需要记录谁参加过什么课程 |
| 保存票的状态 | 票历史 | TicketHistory | 票的状态变迁需要被追踪 |

### 1.3 归纳与筛选

| 候选概念 | 处理 | 理由 |
|---------|------|------|
| CandidateList | 删去 | CandidateList 是 Candidate 的集合，不需要单独的领域概念 |
| Mail | 删去 | 邮件通知由 OA 集成上下文负责，培训上下文不维护邮件概念 |

### 1.4 补充概念

根据培训上下文的完整业务场景（票的分配、票的取消、培训过程），补充以下领域概念：

| 概念 | 英文 | 来源业务场景 |
|------|------|-------------|
| 过滤器 | Filter | 分配票——培训专员设定过滤器过滤员工名单 |
| 有效日期 | ValidDate | 取消票——设置有效日期判断票的有效期限 |
| 有效日期动作 | ValidDateAction | 取消票——不同截止时间的活动策略 |
| 取消动作 | CancellingAction | 取消票——票的取消操作 |
| 出勤 | Attendance | 培训——签到、出勤名单 |
| 黑名单 | Blacklist | 培训——缺勤学员被放入黑名单 |

## 2. 领域分析模型（图 20-51）

### 最终领域概念列表

共 12 个领域概念：

| # | 领域概念 | 英文 | 描述 |
|---|---------|------|------|
| 1 | 培训 | Training | 培训活动的主体，关联课程和教师 |
| 2 | 课程 | Course | 培训要学习的课程，独立维护课程信息 |
| 3 | 票 | Ticket | 系统将分配给员工的培训机会称为"票"，拥有状态 |
| 4 | 票历史 | TicketHistory | 票的状态变迁记录 |
| 5 | 过滤器 | Filter | 用于过滤员工名单，获得提名候选名单 |
| 6 | 有效日期 | ValidDate | 用于判断票的有效期限 |
| 7 | 有效日期动作 | ValidDateAction | 有效日期对应的处理策略 |
| 8 | 取消动作 | CancellingAction | 票的取消操作 |
| 9 | 候选人 | Candidate | 利用过滤器筛选或直接添加的员工 |
| 10 | 出勤 | Attendance | 学员通过二维码签到形成的出勤记录 |
| 11 | 黑名单 | Blacklist | 缺勤学员被放入黑名单 |
| 12 | 学习记录 | Learning | 记录员工参加课程的学习记录 |

### 概念关系

```
Training ──── 包含 ────> Ticket
Training ──── 包含 ────> Candidate
Training ──── 关联 ────> Course
Training ──── 关联 ────> Filter
Training ──── 关联 ────> ValidDate
Training ──── 包含 ────> Attendance
Ticket  ──── 包含 ────> CancellingAction
Ticket  ──── 包含 ────> TicketHistory
ValidDate ── 包含 ────> ValidDateAction
Course  ──── 关联 ────> Learning
Blacklist ── 独立 ────> (无外部关联)
```

## 3. 各概念属性与行为

| 领域概念 | 属性 | 行为 |
|---------|------|------|
| Training | 培训ID、课程、教师、开始日期、结束日期、状态 | 发起培训、分配票、取消培训 |
| Course | 课程ID、名称、描述 | 维护课程信息 |
| Ticket | 票ID、培训ID、状态、拥有者、被提名人ID | 提名、确认、取消 |
| TicketHistory | 票ID、操作类型、状态转换、拥有者、操作者、操作时间 | 记录票状态变更 |
| Filter | 过滤器ID、培训ID、类型、规则 | 过滤员工名单 |
| ValidDate | 有效日期ID、培训ID、公式、日期时间 | 判断票的有效性 |
| ValidDateAction | 动作ID、有效日期ID、动作类型 | 执行有效日期对应的策略 |
| CancellingAction | 动作ID、票ID、原因、动作类型 | 执行取消操作 |
| Candidate | 候选人ID、培训ID、员工ID、姓名 | 作为票的拥有者 |
| Attendance | 出勤ID、培训ID、员工ID、签到时间、签退时间 | 记录出勤 |
| Blacklist | 黑名单ID、员工ID、原因、加入时间 | 加入/移出黑名单 |
| Learning | 学习记录ID、课程ID、员工ID、学习日期 | 检查是否已学习 |
