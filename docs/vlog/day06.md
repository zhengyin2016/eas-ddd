# Day 6 — 培训流程：票的生命周期

今天梳理培训流程，这是三个业务流程中最复杂的一个。培训涉及培训专员、部门协调者、员工主管和员工本人四个角色，引入了一个核心隐喻——"票"(Ticket)，代表分配给员工的培训机会。

培训流程的本质是"分配票"的过程，但它比想象中复杂得多。

**分配票的服务蓝图**：培训专员先设定过滤器过滤员工名单，获得提名候选名单(Candidate)。然后把票分配给部门协调者，协调者再分配给部门员工。员工可以确认或拒绝——如果拒绝，票退回协调者，由协调者再分配，最终形成提名名单(Nomination)。

**培训期间的服务蓝图**：每个学员通过二维码签到，包括培训开始签到和培训结束签到。培训结束后获得出勤名单，与提名名单比较得到缺席名单。确认缺席名单后，系统根据黑名单规则将缺席人员加入黑名单——被列入黑名单的人将来不会再出现在提名候选名单中，除非被移出。

**最复杂的是取消票的流程。** 从发起培训到培训结束有4个截止时间：提名截止、缺席截止、培训开始前、培训结束前。不同截止时间，取消流程完全不同。

以员工取消为例：提名截止前取消需要协调者和主管任一批准；缺席截止前取消只需通知，由培训专员按策略处理票；培训开始前不允许显式取消，未确认的票由系统按策略处理；培训开始后完全不允许取消，只能请假。处理票有3种策略：分享给别的协调者、分配给员工、让票作废。

部门协调者和培训专员也有各自的取消流程，处理策略包括备选名单先到先得、按优先级、手动选择等。

**统一语言方面有几个重要发现。** "分配票给部门协调者"和"分配票给部门员工"虽然都是"分配"，但含义完全不同——协调者获得的是分配资格而非培训资格，员工获得的才是培训资格。所以后者应该叫"提名部门员工"。部门员工在不同阶段有不同身份：候选人(Candidate)是过滤器筛选的员工，具备被提名资格；被提名人(Nominee)是获得票的员工；备选人(Backup)是候选名单减去被提名人后的名单中的员工；学员(Trainee)是被提名人确认参加后的身份。

这些身份不是预先定义的，而是在分析流程的过程中逐步浮现的。统一语言的建立是一个持续的过程。
