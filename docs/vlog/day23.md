# Day 23 - CRM上下文快速建模

**日期**: 第23天
**参与者**: 张毅、郑丽、孙磊
**主题**: CRM上下文领域分析建模

早上刚到办公室，郑丽就抱着笔记本过来了。"张工，HR和PM的建模都已经完成了，今天是不是该轮到CRM了？"

"对，CRM是销售的核心。"我点头，"孙磊今天在吗？"

"在，我刚才叫他了。"郑丽说。

不一会儿，孙磊推门进来。作为销售领域专家，他对CRM业务最熟悉。"张工，听说今天要搞CRM的领域分析？"

"对，用快速建模法。"我打开白板，"郑丽，你昨天做了功课，说说CRM有哪些核心概念？"

郑丽翻了一下笔记："CRM就是客户关系管理，核心流程是：从客户线索开始，转化为商机，商机推进后签合同，然后回款。"

"很好。"我在白板上写下：**客户 → 商机 → 合同 → 回款**。"孙磊，这个流程对吗？"

"基本对。"孙磊点头，"但实际中更复杂。比如一个客户可能有多个商机，一个商机最终对应一个合同，但合同可能分期回款。"

"这很重要。"我说，"这说明我们需要识别出多个领域对象。来，我们用快速建模法——名词提取法。"

大家开始头脑风暴：

**名词提取**：
- 客户（Customer）
- 商机（Opportunity）
- 合同（Contract）
- 回款（Payment）
- 联系人（Contact）

"等等，联系人呢？"郑丽问，"是一个客户可以有多个联系人？"

"对。"孙磊确认，"比如一个企业客户，可能有采购经理、技术总监、财务等多个联系人，我们得区分谁是主要联系人。"

"那客户有等级吗？"我问。

"有啊，A/B/C/D四级。"孙磊说，"A级是VIP客户，交易金额大、频率高；D级是潜力客户。而且等级是动态调整的，不是一成不变的。"

"这个规则很好，说明客户有**升级**和**降级**行为。"我在白板上记录。

"那商机呢？"郑丽问，"商机是怎么推进的？"

"商机有阶段。"孙磊解释，"从初步接触开始，到需求确认、方案报价、商务谈判，最后是合同签订。每个阶段有不同的赢率。"

"赢率？"我问。

"对，初步接触时赢率大约20%，需求确认40%，方案报价60%，商务谈判80%，合同签订95%。"孙磊说得头头是道。

"这个很有价值。"我说，"说明商机阶段是一个值对象，每个阶段对应一个概率值。"

"合同呢？"郑丽问，"合同有哪些状态？"

"合同有生命周期。"孙磊想了想，"草稿、审核中、已审批、执行中、已完成、已终止。合同签订后需要审批，通过后才能执行。"

"那回款呢？"

"回款有计划回款和实际回款。"孙磊说，"合同签订时会制定回款计划，比如分期付款。然后根据实际情况记录回款，可能有逾期。"

我点点头，在白板上整理出了四个聚合：

```
Customer聚合
├── Customer（聚合根）
├── Contact（实体）
└── CustomerLevel、CustomerSource（值对象）

Opportunity聚合
├── Opportunity（聚合根）
└── OpportunityStage（值对象，含赢率）

Contract聚合
├── Contract（聚合根）
├── PaymentPlan（实体）
└── ContractStatus（值对象）

Payment聚合
├── Payment（聚合根）
└── PaymentStatus（值对象）
```

"等等，"郑丽举手，"PaymentPlan和Payment是分开的？"

"对。"我解释，"PaymentPlan是回款计划，属于Contract聚合内部。而Payment是实际的回款记录，独立成聚合根。为什么？因为回款记录可能独立于合同被查询和统计。"

"那合同和商机是什么关系？"孙磊问。

"一个商机可以转化为一个合同。"我在白板上画了箭头，"Contract应该有opportunityId字段指向商机。但这里要注意，这是聚合间的引用，不是包含关系。"

"客户和商机呢？"

"一个客户可以有多个商机。"我说，"Opportunity有customerId字段。"

大家讨论到中午，CRM的领域分析模型基本成型。

"最有价值的聚合是Opportunity。"我总结，"它追踪了从客户需求到签单的完整过程，是销售漏斗的核心。孙磊，你说客户等级是怎么调整的？"

"根据交易金额和频率。"孙磊说，"比如累计交易额超过100万升A级，半年没交易可能降级。"

"这个规则可以放在Customer领域服务里。"郑丽记了下来。

下午，郑丽开始写《CRM领域分析模型》文档，孙磊继续补充业务规则。

看着白板上的四个聚合，我感到满意。CRM上下文的分析建模完成了，接下来是聚合设计。

---

**今日产出**：
- CRM领域分析模型（4聚合、11领域对象）
- 识别出Opportunity为最有价值的聚合
- 明确客户分级策略（A/B/C/D四级）
- 识别商机阶段与赢率对应关系

**关键决策**：
- PaymentPlan属于Contract聚合内部，Payment独立为聚合根
- 客户等级动态调整，基于交易金额和频率
- 商机阶段包含赢率属性
