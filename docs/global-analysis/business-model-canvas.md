# EAS 商业模式画布

> 来源：《解构领域驱动设计》20.3.2 全局分析阶段

## 客户细分

按角色划分：

- **市场部**：管理客户关系，跟踪市场需求
- **项目管理部**：负责项目的流程管理，跟踪项目进度
- **人力资源部**：负责招聘、培训、人事管理
- **集团管理层（决策者）**：需要全局视角掌控人力资源供需状况
- **子公司**：作为集团的组成部分参与人力资源调配
- **项目管理办公室（PMO）**：协调跨项目资源
- **财务**：关注人力成本控制
- **员工**：系统的主要使用者，提交考勤、工作日志等
- **服务中心**：提供支持服务

## 价值主张

- 提高客户关系管理效率
- 项目过程管理的规范化和透明化
- 招聘计划的前瞻性——根据供需预测提前规划
- 通过可视化方式体现人力资源的供需状况
- 避免信息孤岛，实现人力资源的可控

## 渠道

- 由集团下达行政命令在集团内部相关职能部门推广 EAS 系统
- 不需要寻找外部渠道

## 收益来源

- 成本控制和削减——减少人力资源错配带来的浪费
- 为软件交付业务提供更稳定的人力支撑

## 核心资源

- 人力资源——从事软件开发的人力是集团的核心生产力

## 关键业务

- 客户需求管理
- 项目人员管理
- 招聘与培训计划

## 重要合作

- 人力资源招聘有关的外部合作（招聘网站、软件学院）
- 培训有关的合作（培训公司）
