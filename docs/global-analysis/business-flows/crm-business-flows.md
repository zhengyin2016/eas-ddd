## 客户生命周期状态

```
潜在客户 ──→ 活跃客户 ──→ 重要客户 ──→ 战略客户
    │            │            │
    ↓            ↓            ↓
  流失客户     沉默客户     流失客户

客户分级规则：
- A级：当季有明确采购意向，需每日跟进
- B级：有意向但时间不确定，每周跟进
- C级：有潜在需求，每月联系
- D级：一般线索，季度关怀

评级变更：评级由系统根据跟进记录和商机状态动态建议，销售经理确认后生效
```

## 流程1：客户跟进

### 基本信息

| 项目 | 内容 |
|------|------|
| 流程名称 | 客户跟进 |
| 触发条件 | 获取新客户线索 |
| 参与角色 | 销售、销售经理 |
| 周期 | 持续性，根据客户分级决定跟进频率 |

### 业务规则

1. 客户编号自动生成，规则为 `CUS-{YYYY}-{NNNN}`
2. 每个客户有唯一负责人（归属销售），负责人变更需销售经理审批
3. 客户分级（A/B/C/D级），分级由系统建议、销售经理确认
4. 跟进记录需包含：跟进方式、跟进内容、下次跟进计划
5. 客户信息包括：公司名称、行业、规模、联系人（可多个）、地址
6. 联系人信息属于商业机密，仅负责人及其上级可见

### 客户评级动态变更

```
线索进入 → 系统建议评级（D级） → 销售经理确认
    │
    ├─ 有明确采购需求 → 建议升级为A级 → 经理确认
    ├─ 有意向但时间不确定 → 建议升级为B级 → 经理确认
    ├─ 有潜在需求 → 建议升级为C级 → 经理确认
    ├─ 超过30天未跟进 → 建议降级 → 经理确认
    └─ 创建商机 → 自动升级为B级以上
```

## 流程2：商机管理

### 基本信息

| 项目 | 内容 |
|------|------|
| 流程名称 | 商机管理 |
| 触发条件 | 客户表达采购意向 |
| 参与角色 | 销售、销售经理、售前技术 |
| 平均耗时 | 30-180天（从创建到赢单/输单） |

### 业务规则

1. 商机必须关联客户
2. 商机阶段流转：初步接触 → 需求确认 → 方案报价 → 商务谈判 → 合同签订 → 赢单/输单
3. 阶段只能向前推进或标记输单，不能回退
4. 每个阶段有关键动作要求，未完成不允许推进到下一阶段
5. 商机预计金额和预计签单日期随阶段推进逐步精确
6. 输单需填写输单原因（价格、竞争对手、需求变化、其他）

### 关键指标

| 指标 | 定义 | 用途 |
|------|------|------|
| 销售漏斗 | 各阶段的商机数量和金额分布 | 评估销售管线健康度 |
| 赢单率 | 赢单数 / (赢单数 + 输单数) × 100% | 衡量销售效率 |
| 平均销售周期 | 从商机创建到赢单的平均天数 | 预测签单时间 |
| 阶段转化率 | 从当前阶段推进到下一阶段的比例 | 识别销售瓶颈 |

## 流程3：合同签订

### 基本信息

| 项目 | 内容 |
|------|------|
| 流程名称 | 合同签订 |
| 触发条件 | 商机赢单，进入合同签订 |
| 参与角色 | 销售、销售经理、法务、财务 |
| 平均耗时 | 7-30天（从创建到签章生效） |

### 业务规则

1. 合同必须关联客户和商机
2. 合同金额需审批，不同金额不同审批层级：
   - 50万以下：销售经理审批
   - 50万-200万：销售总监审批
   - 200万以上：总经理审批
3. 合同状态流转：草稿 → 审核中 → 已签订 → 执行中 → 已完结
4. 法务审核为必经环节，审核内容包括合同条款合规性、付款条件、违约责任
5. 合同签章需双方盖章确认，电子签章与纸质签章具有同等效力
6. 合同归档后生成唯一的合同编号

### 合同金额审批层级

```
合同金额 < 50万   → 销售经理审批
合同金额 < 200万  → 销售总监审批
合同金额 ≥ 200万  → 总经理审批

审批流程：
销售提交 → 金额判断 → 对应审批人 → 审批通过/驳回
                                          │
                              通过后 → 法务审核（必经）
                                          │
                              法务通过 → 客户确认 → 签章生效
```

## 流程4：回款跟踪

### 基本信息

| 项目 | 内容 |
|------|------|
| 流程名称 | 回款跟踪 |
| 触发条件 | 合同签订后的回款计划 |
| 参与角色 | 销售、财务、销售经理 |
| 周期 | 持续性，按回款计划节点跟踪 |

### 业务规则

1. 回款必须关联合同和回款计划
2. 回款计划在合同签订时制定，可按比例分期（如3-3-3-1或5-4-1）
3. 回款到期前7天自动提醒销售，到期当日再次提醒
4. 逾期自动预警，超过30天升级至销售经理
5. 回款由财务确认入账，销售不能直接确认
6. 回款进度影响客户信用评级

### 回款逾期升级机制

```
到期前 7天 → 提醒销售跟进
到期当日  → 再次提醒销售
逾期 1-7天 → 逾期标记，每日提醒销售
逾期 8-30天 → 预警升级至销售经理
逾期 31-60天 → 预警升级至销售总监
逾期 >60天 → 标记为坏账风险，上报总经理
```

| 流程 | 交互上下文 | 交互类型 | 数据流向 | 说明 |
