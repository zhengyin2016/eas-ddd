# 业务服务识别文档

> **项目**：EAS（企业自动化管理系统）
> **阶段**：全局分析阶段 — 业务服务识别
> **日期**：Day 6
> **作者**：张毅

---

## 1. 概述

本文档记录EAS系统三个核心业务领域（HR、PM、CRM）的业务服务识别结果。业务服务通过以下方法从业务流程中提炼：

- **业务场景**：每个业务流程的"前台触点"构成一个业务场景
- **业务服务**：每个业务场景中直接贡献于业务价值的后台操作步骤

### 识别原则

1. **核心业务服务**必须直接贡献于业务价值
2. **支撑功能**（如消息通知、日志记录）不列为业务服务，归入支撑子领域
3. 业务服务粒度以"可独立归属限界上下文"为准
4. 相似操作在不同场景下出现时，合并为同一业务服务

## 2. HR 业务服务

> **领域**：人力资源管理（Human Resource Management）
> **业务服务数量**：12

| 编号 | 业务场景 | 业务服务 | 描述 |
|------|---------|---------|------|
| HR-01 | 员工入职 | 创建员工 | HR录入员工基本信息，包括姓名、身份证号、联系方式等 |
| HR-02 | 员工入职 | 分配组织 | 将员工关联到指定部门和岗位，建立组织关系 |
| HR-03 | 员工离职 | 办理离职 | 处理员工离职手续，包括离职原因记录、交接事项确认 |
| HR-04 | 员工调岗 | 办理调岗 | 处理员工岗位变更，更新部门归属和岗位信息 |
| HR-05 | 招聘需求 | 提交招聘需求 | 用人部门提交人员需求，包括岗位、人数、技能要求 |
| HR-06 | 招聘流程 | 安排面试 | HR安排候选人面试，包括时间、面试官、面试形式 |
| HR-07 | 考勤管理 | 记录考勤 | 记录员工打卡信息，支持异常考勤处理 |
| HR-08 | 培训管理 | 创建培训计划 | HR制定培训计划，包括课程、讲师、时间安排 |
| HR-09 | 培训管理 | 培训报名 | 员工报名参加培训课程 |
| HR-10 | 技能管理 | 维护技能档案 | 更新员工技能信息和资质认证记录 |
| HR-11 | 储备人才 | 管理储备人才 | 维护人才库信息，跟踪候选人的意向和状态 |
| HR-12 | 员工查询 | 查询可用员工 | PM查询当前可分配的员工，按技能和可用时间筛选 |

## 3. PM 业务服务

> **领域**：项目过程管理（Project Management）
> **业务服务数量**：10

| 编号 | 业务场景 | 业务服务 | 描述 |
|------|---------|---------|------|
| PM-01 | 项目立项 | 创建项目 | PM创建新项目，定义项目基本信息、目标和时间范围 |
| PM-02 | 项目立项 | 审批项目 | PMO审批项目立项申请，确认项目可行性 |
| PM-03 | 人员分配 | 分配项目成员 | PM将可用员工分配到项目，指定角色和参与时段 |
| PM-04 | 人员分配 | 查询可用资源 | 查询当前可分配的人力资源，按技能、部门、时间筛选 |
| PM-05 | 任务管理 | 创建任务 | PM分解项目任务，定义任务描述、负责人、截止时间 |
| PM-06 | 进度跟踪 | 更新任务状态 | 项目成员更新任务进度和状态（待开始/进行中/已完成） |
| PM-07 | 问题管理 | 创建Issue | 记录项目执行过程中的问题和风险 |
| PM-08 | 问题管理 | 解决Issue | 处理并关闭项目问题，记录解决方案 |
| PM-09 | 项目结项 | 结项审批 | PMO审批项目结项，确认交付物和项目总结 |
| PM-10 | 项目查询 | 查询项目进度 | 查看项目整体进度、资源使用情况和风险状态 |

## 4. CRM 业务服务

> **领域**：客户关系管理（Customer Relationship Management）
> **业务服务数量**：10

| 编号 | 业务场景 | 业务服务 | 描述 |
|------|---------|---------|------|
| CRM-01 | 客户跟进 | 创建客户 | 销售人员录入客户基本信息和联系方式 |
| CRM-02 | 客户跟进 | 更新客户 | 维护客户信息，包括跟进记录、客户状态变更 |
| CRM-03 | 商机管理 | 创建商机 | 从客户需求中识别并创建销售商机 |
| CRM-04 | 商机管理 | 推进商机阶段 | 更新商机在销售漏斗中的阶段（初步接触/需求确认/方案报价/商务谈判/赢单/输单） |
| CRM-05 | 合同管理 | 创建合同 | 商机赢单后生成合同，定义合同金额、条款和履约计划 |
| CRM-06 | 合同管理 | 审核合同 | 内部审核合同内容的合规性和完整性 |
| CRM-07 | 回款管理 | 制定回款计划 | 合同签订后制定分笔回款的时间节点和金额 |
| CRM-08 | 回款管理 | 记录回款 | 记录实际回款金额和时间，跟踪回款进度 |
| CRM-09 | 客户查询 | 查询客户信息 | 搜索和查看客户详情，包括历史交易和跟进记录 |
| CRM-10 | 销售分析 | 分析销售漏斗 | 统计商机转化率、赢单率等销售指标 |

## 6. 跨领域业务服务关联

以下业务服务存在跨领域依赖关系：

| 上游服务 | 下游服务 | 关联说明 |
|---------|---------|---------|
| HR-12 查询可用员工 | PM-03 分配项目成员 | PM需要查询HR领域的员工信息来完成人员分配 |
| HR-12 查询可用员工 | PM-04 查询可用资源 | 资源查询依赖HR领域的员工数据 |
| CRM-05 创建合同 | PM-01 创建项目 | 商机赢单后的合同可能触发新项目的创建 |
| CRM-10 分析销售漏斗 | PM-10 查询项目进度 | 销售分析可能需要参考项目执行情况 |

## 7. 被排除的功能（非核心业务服务）

以下功能在识别过程中被讨论，但最终排除在核心业务服务之外：

| 功能 | 排除原因 | 归属 |
|------|---------|------|
| 发送入职通知 | 不直接贡献于业务目标，去掉不影响入职完成 | 支撑子领域（消息通知） |
| 发送面试通知 | 同上，通知为辅助动作 | 支撑子领域（消息通知） |
| 导出报表 | 通用功能，非业务核心 | 支撑子领域 |
| 审批流配置 | 系统配置功能，非业务服务 | 通用子领域 |
