# EAS 价值需求

> 来源：《解构领域驱动设计》20.3.2 全局分析阶段

## 1. 利益相关者

| 编号 | 利益相关者 | 关注点 |
|------|-----------|--------|
| 1 | 集团决策者 | 人力资源供需平衡的全局掌控 |
| 2 | 子公司 | 参与集团人力资源调配 |
| 3 | 人力资源部 | 招聘、培训、人事管理 |
| 4 | 市场部 | 客户关系管理、市场需求 |
| 5 | 项目管理部 | 项目流程管理、项目进度跟踪 |
| 6 | 项目管理办公室（PMO） | 跨项目资源协调 |
| 7 | 财务 | 人力成本控制 |
| 8 | 员工 | 考勤、工作日志等日常工作 |
| 9 | 服务中心 | 支持服务 |

## 2. 系统愿景

> 避免信息孤岛，实现人力资源的可控，从而达到人力资源的供需平衡。

## 3. 系统范围

### 3.1 当前状态（AS-IS）

1. 创建了由项目经理、业务分析师、开发人员和测试人员构成的特性团队
2. 集团项目管理部负责项目的流程管理
3. 集团已有 OA 系统作为部门之间的流程协作与消息通知
4. 集团已制订了人才招聘管理办法、项目过程管理办法
5. 软件学院和招聘网站的简历作为集团的人才储备库
6. 员工培训已有合作的培训公司
7. 市场部提供客户和潜在客户名单
8. 由集团下达行政命令在集团内部相关职能部门推广 EAS 系统

### 3.2 未来状态（TO-BE）

1. EAS 系统通过用户验收
2. EAS 系统上线运行
3. 由 EAS 系统负责客户关系管理、项目管理、人力资源管理
4. 通过客户满意度评估 EAS 系统的价值

### 3.3 目标列表

1. 通过可视化方式体现人力资源的供需状况
2. 管理集团与客户和潜在客户之间的关系，管理市场需求，对合同进行跟踪
3. 管理项目和项目人员，跟踪项目进度
4. 实时调整用人需求，制订招聘和培训计划
5. 管理员工考勤、工作日志等日常工作
