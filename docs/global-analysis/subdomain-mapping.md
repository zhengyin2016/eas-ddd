# 子领域划分

> 本文档记录 EAS 系统的子领域划分结果。
> 来源：《解构领域驱动设计》第 20.3.2 节
> 划分策略：从痛点与价值出发，按功能分类

---

## 划分策略

功能分类策略，从痛点与价值出发。每一个子领域的划分都对应着EAS系统核心诉求列表中的痛点或价值点。核心子领域直接产生业务价值，通用子领域为所有业务流程提供基础支撑，支撑子领域辅助核心领域的执行活动。

---

## 核心子领域（7个）

核心子领域是EAS系统直接产生业务价值的部分，每个都对应着痛点列表中的核心诉求。

| 编号 | 子领域 | 业务价值 |
|------|--------|---------|
| 1 | 决策分析 | 统计报表为管理决策提供支持 |
| 2 | 市场需求管理 | 管理市场需求、客户需求、订单 |
| 3 | 客户关系管理 | 客户关系管理效率（含合同管理） |
| 4 | 员工管理 | 员工日常工作管理（含考勤、工作日志） |
| 5 | 人才招聘 | 招聘计划前瞻性（含储备人才） |
| 6 | 技能培训 | 培训计划制定与执行 |
| 7 | 项目进度管理 | 项目过程管理 |

---

## 通用子领域（2个）

通用子领域是所有业务流程都依赖的基础设施，不产生差异化价值但不可或缺。

| 编号 | 子领域 | 业务价值 |
|------|--------|---------|
| 1 | 组织结构管理 | 保证业务流程运转 |
| 2 | 认证授权 | 访问安全 |

---

## 支撑子领域（3个）

支撑子领域为核心领域的执行活动提供支持，与核心竞争力关系较弱。

| 编号 | 子领域 | 业务价值 |
|------|--------|---------|
| 1 | 资源分配 | 服务中心硬件资源分配（与供需平衡无直接关系） |
| 2 | 消息通知 | 支持大部分核心领域的执行活动 |
| 3 | 文件上传下载 | 支持大部分核心领域的执行活动 |

## 子领域与业务场景的对应关系

| 子领域 | 类型 | 对应业务场景 |
|--------|------|------------|
| 决策分析 | 核心 | 项目统计报表 |
| 市场需求管理 | 核心 | 市场需求管理、需求订单管理 |
| 客户关系管理 | 核心 | 合同管理、简历管理 |
| 员工管理 | 核心 | 员工考勤、工作日志 |
| 人才招聘 | 核心 | 储备人才管理 |
| 技能培训 | 核心 | 票的分配、票的取消、培训签到 |
| 项目进度管理 | 核心 | 项目管理、项目成员管理、项目计划、问题管理 |
| 组织结构管理 | 通用 | 部门、团队组织 |
| 认证授权 | 通用 | 身份认证、权限控制 |
| 资源分配 | 支撑 | 硬件资源分配 |
| 消息通知 | 支撑 | 邮件通知、系统通知 |
| 文件上传下载 | 支撑 | 合同归档、文件存储 |
