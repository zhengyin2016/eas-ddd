# 统一语言词典 (Ubiquitous Language Dictionary)

**编号**: EAS-UL-001
**版本**: 1.0
**日期**: 2026-05-29
**作者**: 张毅

## 概述

本文档定义EAS系统的统一语言词典。统一语言是DDD的核心实践，确保业务专家和开发团队使用一致的术语。

**重要**：统一语言是**限界上下文级别**的，同一术语在不同上下文可能有不同含义。

---

## 全局术语

| 中文 | English | 定义 | 示例 |
|-----|---------|------|------|
| 限界上下文 | Bounded Context | 系统的边界，内部有统一的业务语义 | HR上下文、PM上下文、CRM上下文 |
| 子领域 | Subdomain | 业务问题空间的划分 | 核心子领域、通用子领域、支撑子领域 |
| 聚合 | Aggregate | 领域模型中的一致性边界 | 员工聚合、项目聚合 |
| 聚合根 | Aggregate Root | 聚合的唯一入口 | Employee、Project |
| 值对象 | Value Object | 不可变的、通过属性标识的对象 | Email、地址、金额 |
| 领域事件 | Domain Event | 领域内已发生的事实 | 员工已入职、合同已签订 |
| 上下游 | Upstream/Downstream | 上下文间的依赖关系 | PM是HR的下游 |

## HR上下文术语

| 中文 | English | 定义 | 示例 |
|-----|---------|------|------|
| 员工 | Employee | 与公司签订劳动合同的自然人，有工号、所属组织、岗位 | 张三(E001)，研发部，高级工程师 |
| 部门 | Department | 组织的基本单元，隶属于某个组织 | 研发部、市场部 |
| 岗位 | Position | 职位定义，包含职责、职级要求 | 高级工程师、产品经理 |
| 职级 | Level | 员工的专业等级，决定薪酬范围 | L1~L10，L5为高级工程师 |
| 员工状态 | Employee Status | 员工当前的雇佣状态 | 在职(ACTIVE)、离职(RESIGNED)、请假中(ON_LEAVE) |
| 招聘需求 | Recruitment Requirement | 部门提出的人员招聘请求，需审批 | 招聘Java开发5名，紧急 |
| 面试 | Interview | 招聘流程中的候选人面试安排 | 李四的Java工程师面试，2026-05-30 |
| 考勤 | Attendance | 员工的出勤记录统计 | 张三5月出勤22天，请假1天 |
| 培训计划 | Training Plan | 组织的培训活动安排 | DDD内训，2026-06-01，30人 |
| 技能 | Skill | 员工掌握的专业技能及水平 | Java(专家)、DDD(熟练) |
| 储备人才 | Talent Pool | 具有潜力但当前无合适位置的人才储备 | 管培生库、候选人池 |
| 全职 | Full-time | FTE为1.0，全职投入项目 | 项目需要5名全职开发 |
| 兼职 | Part-time | FTE小于1.0，部分时间投入项目 | 项目需要0.5个测试 |

## PM上下文术语

| 中文 | English | 定义 | 示例 |
|-----|---------|------|------|
| 项目 | Project | 为交付软件产品而组织的临时性 effort，有明确目标和时间边界 | EAS企业协作系统项目，2026-05-01至2026-07-31 |
| 项目状态 | Project Status | 项目当前所处的阶段 | 待审批(PENDING)、进行中(IN_PROGRESS)、已完成(COMPLETED) |
| 里程碑 | Milestone | 项目中的重要时间节点或交付节点 | 需求冻结、上线发布 |
| 任务 | Task | 项目中需要完成的具体工作项 | 实现登录功能 |
| Issue | Issue | 项目执行中发现的问题或缺陷 | 登录接口500错误 |
| 迭代 | Iteration/Sprint | 项目执行的时间盒子，通常1-2周 | Sprint 1，2026-05-01至2026-05-14 |
| 人员分配 | Project Assignment | 将员工分配到项目，指定角色和投入度 | 张三以50%FTE担任项目经理 |
| 全职比例 | FTE Ratio | 员工在项目中的时间投入比例，1.0表示全职 | 0.5表示半天时间投入项目 |
| Story Point | Story Point | 任务复杂度的估算单位 | 这个任务是3个Story Point |
| 优先级 | Priority | 任务或Issue的处理优先级 | P0紧急、P1高、P2中、P3低 |

## CRM上下文术语

| 中文 | English | 定义 | 示例 |
|-----|---------|------|------|
| 客户 | Customer | 有业务往来的公司或个人 | 某科技公司 |
| 客户等级 | Customer Level | 客户的重要性分级 | A级(战略)、B级(重点)、C级(一般) |
| 联系人 | Contact | 客户方的对接人信息 | 王总，CTO，13800000001 |
| 商机 | Opportunity | 潜在的销售机会 | EAS系统销售机会，预计50万 |
| 商机阶段 | Opportunity Stage | 商机当前所处的销售阶段 | 线索→需求→方案→谈判→赢单 |
| 销售漏斗 | Sales Funnel | 商机在各阶段的分布与转化分析 | 本月商机100个，赢单率20% |
| 合同 | Contract | 与客户签订的正式法律文件 | EAS系统开发合同，50万 |
| 合同状态 | Contract Status | 合同的审批/执行状态 | 草稿→审批中→已签→执行中→已完成 |
| 回款 | Payment | 客户按合同支付的款项 | 首付款15万，2026-06-01到账 |
| 回款计划 | Payment Plan | 合同约定的分期付款计划 | 签约30%、交付50%、验收20% |

## 组织上下文术语

| 中文 | English | 定义 | 示例 |
|-----|---------|------|------|
| 组织 | Organization | 公司或分支机构的基本单位 | 总公司、上海分公司 |
| 部门 | Department | 组织下属的职能单元 | 研发部、市场部、人力资源部 |
| 组织树 | Organization Tree | 组织的层级结构表示 | 总公司→研发部→前端组 |
| 上级部门 | Parent Department | 部门在组织树上的上级 | 研发部的上级是总公司 |
| 岗位 | Position | 职位的定义，不关联具体人 | 高级工程师岗位，职级L5 |

## 跨上下文术语差异

### "项目"在不同上下文的含义

| 上下文 | 含义 | 关键属性 |
|-------|------|---------|
| PM上下文 | 软件交付项目 | 需求、迭代、交付物、团队成员 |
| CRM上下文 | 商机跟进项目 | 商机、阶段、赢单率、销售活动 |

### "合同"在不同上下文的归属

| 上下文 | 对"合同"的职责 |
|-------|---------------|
| CRM上下文 | 合同签订流程、合同信息维护 |
| PM上下文 | 关联项目到合同、基于合同创建项目 |
| 财务系统 | 开票、回款认领 |

## 使用指南

### 开发团队

1. 代码命名统一使用词典中的英文术语
2. 领域模型类名使用聚合根名称
3. API接口使用资源名称的复数形式

### 业务团队

1. 需求文档使用统一的中文术语
2. 跨上下文讨论时明确上下文
3. 新术语需要更新词典
